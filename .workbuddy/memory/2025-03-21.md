### 核心发现
1. **双通道架构**
   - 订单通道 (`/ws/order/{client_id}`): 专门处理订单和撤单结果推送
   - 行情通道 (`/ws/quote/{client_id}`): 专门处理实时行情数据推送

2. **统一连接管理**
   - `WebSocketManager` 统一管理所有 WebSocket 连接
   - 支持同一客户端多连接（如多标签页）
   - 自动清理失效连接

3. **订单推送流程**
   - HTTP 请求 → 订单 API → QMT 交易接口 → WebSocket 推送
   - 下单和撤单都会实时推送结果到对应账户的 WebSocket 连接

4. **行情推送流程**
   - WebSocket 订阅 → 注册回调 → xtquant 推送数据 → 回调处理 → 广播到所有订阅客户端
   - 使用 `asyncio.run_coroutine_threadsafe()` 处理同步回调到异步广播的转换

5. **消息标准化**
   - 统一消息格式: `{type, channel, data, timestamp, client_id}`
   - 支持自动转换为标准化格式

### 技术要点
1. **异步回调处理**: 保存主事件循环引用，使用 `run_coroutine_threadsafe` 在同步回调中调度异步任务
2. **频道过滤**: 广播时只发送给订阅了特定频道的客户端
3. **数据验证**: 行情回调中对数据进行类型检查、字段验证、数值范围检查
4. **心跳机制**: 30秒超时自动发送心跳，支持客户端 ping/pong
